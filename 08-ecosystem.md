## Роль Wi‑Fi (Ruckus Unleashed)

| AP (имя в кластере) | Модель | Зона (Description) | Источник |
|---------------------|--------|-------------------|----------|
| **ch-int-ap-01** | R550 | Dining Room | `Ruckus.conf` |
| **ch-int-ap-02** | R550 | Living Room L | `Ruckus.conf` |
| **ch-int-ap-03** | R350 | Living Room F | `Ruckus.conf` |
| **ch-int-ap-04** | R350 | Living Room C | `Ruckus.conf` |
| **ch-ext-ap** | T350D | Bathhouse (outdoor) | `Ruckus.conf` |

**Итог по числу AP:** 2×R550 + 2×R350 внутри дома, 1×T350D снаружи (баня/участок) — совпадает с планом из экспорта.

**Wi‑Fi и «нативный» IoT на Ruckus:** точки доступа **не** расширяют радиоконтур Zigbee/Matter/Thread сторонних хабов (например Aqara M3). Для покрытия участка (теплица и т.д.) нужны **свои** ретрансляторы сети хаба (Zigbee/Thread), LoRaWAN (см. Lavritech) или отдельные шлюзы — см. обсуждение в экспорте.

**Встроенный Zigbee R550:** в экспорте зафиксирован вывод, что в режиме **Unleashed** штатный Zigbee на R550 **не используется** как замена внешнему хабу; опора на **Aqara M3** (и аналоги). Региональный код в `Ruckus.conf` — **Russia**; любые ограничения по радиомодулям на стороне вендора и прошивки **перепроверять** по актуальной документации Ruckus для вашей партии AP.

## Контуры автоматизации и облака

| Слой | Роль | Примечание |
|------|------|------------|
| **Home Assistant** (2026.x) | Основная логика, UI, интеграции | В сети проекта — хост **`ch-smart`** VLAN 10 (`01-network.md`) |
| **Wiren Board 8.x** | Полевая автоматика, **центральный MQTT-брокер** | `ch-wirenboard`, ZigBee-модуль **WBE2R-R-ZIGBEE** (`07-wirenboard.md`); не путать с маркетинговыми названиями из сторонних чатов |
| **Lavritech L2 Gate** | LoRaWAN → MQTT на брокер WB | `07-wirenboard.md` |
| **Aqara M3** | Matter / Thread BR / Zigbee для экосистемы Aqara | Дом; согласование с HA (Matter: избегать `localhost` из Docker — использовать `network_mode: host` или IP хоста) |
| **Apple Home** | Потребительский клиент | Из экспорта |
| **Samsung SmartThings** | Только управление **кондиционером** | Уточнение из экспорта: не основной контур сад/теплица |
| **OpenRouter** | Доступ к LLM (в т.ч. DeepSeek) из HA | Реализовано по экспорту |
| **Tuya (Zigbee)** | Управление **газовым котлом Kiturami** и связанным **тёплым полом** | Уточнение: не «Tuya Wi‑Fi розетки участка»; Wi‑Fi Tuya из общей картины убраны |

## Ворота (откатные), зона **Cabin** (бытовка)

**Физика и привод:** привод **Nice Robo 600 KCE**, контроллер **R0A41**, импульс через **Aqara Dual Relay Module T2** (сухой контакт) — см. также экспорт обсуждения. Ворота относятся к **бытовке (Cabin)**, а не к жилому дому.

**Каналы управления (параллельно, не исключают друг друга):**

| Канал | Назначение |
|-------|------------|
| **Пульт Nice** | Штатное локальное/радио управление приводом |
| **Aqara** | Сухой контакт (T2) / сценарии через хаб **M3** |
| **Hikvision AX PRO** | Охранная система: сценарии/реле/автоматизация в рамках конфигурации AX PRO (панель **DS-PWA96**, `10.254.50.18`, **Boiler Room** — см. `03-security.md`) |
| **Home Assistant** | Логика и объединение; HA связан с **Aqara по Matter**, оттуда же цепочка в **Apple Home** — команды к воротам приходят через тот же контур, что и остальная Aqara-автоматика |

**Важно:** клеммы и параметры привода (например F5.2) — только по **официальной инструкции** Nice для вашей ревизии; в репозитории нет электрической спецификации.

## Что не переносить из чата «как факт»

- Модели железа Wiren Board, названные в чате иначе, чем в `wb-hardware.conf` / `07-wirenboard.md`.
- Любые **пароли** и PSK (в т.ч. гостевая сеть): хранить вне git или в секрет-хранилище.
- Однозначные юридические/региональные утверждения о «блокировке» модулей без ссылки на bulletin Ruckus.
