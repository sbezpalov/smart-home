# Безопасность репозитория

## Что считается секретом

- Пароли и хэши учётных записей на сетевом оборудовании.
- Wi‑Fi **PSK / SAE**, RADIUS **shared secret**, SNMP **community** (даже `public`/`private` — повод пересмотреть политику).
- Блоки **`ENC …`**, **`password encrypted`**, **`BEGIN … PRIVATE KEY`** в выгрузках FortiGate / MikroTik / Cisco и т.п.
- Любые токены Notion, API, облака — только в переменных окружения или менеджере секретов, не в файлах репозитория.

## Что сделано в этом репозитории

- В **`.gitignore`** указано `sources/*.conf`: живые конфиги остаются у вас в `sources/` на диске, но **не должны попадать в новые коммиты** (после `git rm --cached` они снимаются с индекса).
- В локальной **`sources/Ruckus.conf`** открытый текст PSK/SAE заменён на плейсхолдер `********` (на устройстве пароли не менялись — при следующей выгрузке снова появятся в файле; не коммитьте их).
- Каталог **`sources/`** в git удерживается файлом **`sources/.gitkeep`**.

## Обязательные действия при утечке или публичном remote

1. **Сменить** скомпрометированные пароли, PSK Wi‑Fi, SNMP community, VPN PSK — всё, что могло оказаться в истории коммитов.
2. Убедиться, что **remote только приватный** (GitHub/GitLab private, self‑hosted). Публичный репозиторий с бэкапами конфигов недопустим.
3. При необходимости **очистить историю** (утёкший коммит уже запушен): `git filter-repo` или BFG Repo-Cleaner по путям вроде `sources/*.conf` и `**/Ruckus.conf`, затем force-push и ротация секретов.

## Рекомендуемый рабочий процесс

- Держать **приватную** копию полных `*.conf` вне git (архив, зашифрованный том, корпоративный vault).
- В git при необходимости — только **обезличенные** фрагменты или отдельный приватный репозиторий только для бэкапов.
- Для Cursor/агентов: не вставлять в чат полные дампы с секретами; для документации в Notion — без паролей (как уже делалось для Wi‑Fi).
